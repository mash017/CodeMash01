React Props vs State

In React, props and state are two important concepts for managing data in components.

Props (Properties)

  • Passed from a parent component to a child component.
  • Read-only â€” cannot be modified by the child component.
  • Used for dynamic data and configuration.

State

  • Managed within the component itself.
  • Can be updated using setState or the useState Hook.
  • Used for data that changes over time.

Example

import React, { useState } from 'react';

function Greeting({ name }) {
  const [count, setCount] = useState(0);

  return (
    <div>
      <h1>Hello, {name}!</h1>
      <p>You clicked {count} times</p>
      <button onClick={() => setCount(count + 1)}>Click Me</button>
    </div>
  );
}

// Usage
<Greeting name="Mash" />

Key Differences:

Props State
Immutable Mutable
Passed from parent Managed within component
Static for the component Can change dynamically
